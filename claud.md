# Préférence utilisateur (workflow)

Ce projet est maintenu pour un utilisateur non technique.

## Règle de fonctionnement demandée

- Si la demande est claire et faisable, l'agent doit **coder directement**.
- Ne pas attendre une confirmation supplémentaire du type "tu peux coder".
- Limiter les explications longues avant action; privilégier l'exécution.
- Poser une question uniquement si une information est bloquante.
-proposer les ajustements nécessaires sur supabase lorsque c'est pertinent

## Style attendu

- Réponse simple, concrète, en français.
- Priorité au résultat visible dans le produit.
